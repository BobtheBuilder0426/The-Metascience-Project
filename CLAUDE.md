## What this folder is
Project **the Metascience Project -- "Built with Claude: Life Sciences" (Researcher Track).** Mission: use **Claude Science** to discover
the best automated, self-improving research loop -- the **Agentic Loop** (Claude Code drives Claude Science in a closed
cycle) -- and prove, measurably, that it beats a raw baseline, written up publication-style. This folder is the joint
Claude Science + Claude Code workspace **and** the open-source repo we submit.

1. **`STATUS.md`** -- where we are right now + the single next action. (Read FIRST on any resume.)
2. **`DOCUMENTATION.md`** -- how this repo works: structure, roles/tags, the loop, naming, documentation duties.
3. **`GOAL.md`** -- the locked target (Northstar + Sections 1-6).
Then as needed: `ROADMAP.md` (the phase plan) - `resources/` (what you may use: the Claude Science handbook, RESOURCES.md,
the starter papers) - `LABBOOK.md` (the full history).

## How to work here (the rules)
- **The LABBOOK (`LABBOOK.md`) is the highest truth** -- everything that happens MUST be traceable in it. Append an
  entry after every unit of work; never rewrite past entries; if two docs ever disagree, the labbook wins.
- **Update `STATUS.md` after every step** and at each phase transition.
- **Register every source in `SOURCES.md`** (`S-NNN`) and cite it by number.
- **English everywhere.** Every file begins with a crash-recovery header (what it is + how to use it).
- **Naming (loose files):** `YYYY-MM-DD_HHMM_<category>_<short-description>.<ext>` (details in `DOCUMENTATION.md`).
- **Never run heavy code in-place** from this folder -- pull it to a local runtime area, run there, copy results back.
- **Hand-offs:** `STATUS.md`'s "Next action" is the baton -- update it + the labbook at every hand-off (see `DOCUMENTATION.md` section 4b).
- Full detail for all of the above is in **`DOCUMENTATION.md`**.
